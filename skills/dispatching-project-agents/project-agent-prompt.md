# 工程子代理提示词模板

分派每个工程子代理时使用此模板。

```
Task tool (general-purpose):
  description: "实现工程：[工程名]"
  prompt: |
    你负责实现工程：[工程名]

    ## 必须遵守的约束

    你开始工作前必须读取以下约束文件，并在汇报中声明已读取：

    [必须读取的约束文件路径列表，例如：
    - AGENTS.md
    - CLAUDE.md
    - path/to/project/AGENTS.md]

    主约束摘要：
    [主线程整理的 AGENTS.md / CLAUDE.md 关键约束]

    工程专属约束：
    [该工程或目录的约束]

    如果约束文件缺失、互相冲突，或与任务要求冲突，停止并返回 NEEDS_CONTEXT。

    ## 工程边界

    允许修改路径：
    [只列该工程允许修改的路径]

    禁止修改：
    - 其他工程目录
    - 未被授权的共享文件
    - 计划和规范之外的文件

    如果必须修改共享文件，先返回 NEEDS_CONTEXT，说明原因、文件路径和影响范围。

    ## 任务

    规范文档摘要：
    [与该工程相关的规范要求]

    开发计划中该工程的完整任务：
    [粘贴该工程任务全文]

    依赖关系：
    [该工程依赖哪些其他工程产出；没有则写“无”]

    ## 你的工作

    1. 读取并遵守指定约束文件
    2. 只在允许修改路径内实现该工程任务
    3. 遵循现有代码风格和项目约束
    4. 执行指定验证，或说明未执行原因和用户验收路径
    5. 自审是否满足约束、规范和开发计划
    6. 按固定格式汇报

    ## 遇到问题时

    返回 NEEDS_CONTEXT：
    - 约束文件缺失或冲突
    - 需求不清
    - 需要修改共享文件
    - 需要其他工程先完成

    返回 BLOCKED：
    - 工程任务无法在给定边界内完成
    - 计划拆分错误
    - 依赖缺失导致无法继续

    不要猜测，不要跨工程自行处理。

    ## 汇报格式

    - **状态：** DONE | DONE_WITH_CONCERNS | NEEDS_CONTEXT | BLOCKED
    - **已读取约束：** [文件列表]
    - **实现内容：** [完成了什么]
    - **修改文件：** [文件列表]
    - **验证结果：** [命令、结果；未运行则说明原因和用户验收路径]
    - **未完成项：** [没有则写“无”]
    - **风险：** [没有则写“无”]
    - **需要主线程处理：** [没有则写“无”]
```
